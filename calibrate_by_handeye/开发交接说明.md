# 手眼标定开发交接说明

这份文档面向后续开发者，目标不是教最终用户操作，而是帮助快速理解当前手眼标定链的入口、数据流、坐标约定和常见坑点。

## 目录内角色

- `start_handeye_session.py`
  - 连续手动采样入口。
  - 打开相机、显示 ChArUco 检测结果、持续打印关节角，按 `s` 保存样本。
- `collect_handeye_sample.py`
  - 单次采样入口。
  - 适合脚本化采一条样本，底层复用 `start_handeye_session.py` 的运行时和保存逻辑。
- `solve_eye_to_hand.py`
  - 从 `sample.json` 集合求解 `T_base_camera`。
  - 默认会优先解析 `samples/` 下时间最新的 `session_*`。
- `validate_eye_to_hand.py`
  - 用“反推 `T_ee_board` 是否恒定”来检查当前外参是否自洽。

## 数据流

当前链路是：

1. `start_handeye_session.py` 或 `collect_handeye_sample.py`
2. 在 `samples/session_xxx/sample_yyy/sample.json` 中保存：
   - `board_detection`
   - `joint_state`
   - `ee_reference.T_base_ee`
3. `solve_eye_to_hand.py`
   - 读取所有有效样本
   - 求解 `extrinsics_eye_to_hand.json`
4. `validate_eye_to_hand.py`
   - 重新读取样本和 `extrinsics_eye_to_hand.json`
   - 反推每条样本的 `T_ee_board`
   - 统计平移/旋转离散程度

## 当前统一坐标约定

本目录统一采用：

- `T_x_y` 表示“把 `y` 坐标系中的点变换到 `x` 坐标系”
- 即：

```text
p_x = T_x_y * p_y
```

当前几个最重要的量：

- `T_base_ee`
  - 末端坐标系到机械臂基座坐标系
  - 来自实时关节角 + URDF 正运动学
- `T_camera_board`
  - 标定板坐标系到相机坐标系
  - 来自 `solvePnP` / `detect_charuco`
- `T_base_camera`
  - 相机坐标系到机械臂基座坐标系
  - 手眼标定最终要求的外参
- `T_ee_board`
  - 标定板坐标系到末端坐标系
  - 理想情况下在整批样本中应保持常量

验证时使用的核心关系：

```text
T_ee_board = inv(T_base_ee) @ T_base_camera @ T_camera_board
```

如果这条式子在所有样本里反推出的 `T_ee_board` 很稳定，就说明当前 `T_base_camera` 和样本约定基本一致。

## validate 的判断标准

`validate_eye_to_hand.py` 的核心思想不是“再求一次外参”，而是看：

- 同一块板固定在末端之后
- 不同样本反推出来的 `T_ee_board`
- 是否近似不变

结果里最重要的是：

- `translation_error_stats_m`
- `rotation_error_stats_deg`

经验上：

- 平移均值在厘米级以内，通常说明样本和外参比较自洽
- 旋转均值在几度以内，通常说明装夹和求解结果比较稳定

注意这里的“绝对平移大小”不一定小：

- 如果板子有夹具、配重、加高结构
- `T_ee_board` 的平移分量本身可以比较大

真正要看的是：

- “样本间离散度”是否小
- 不是“绝对值是否看起来小”

## 已知限制和历史坑点

- 历史上出现过坐标方向约定混淆，尤其是：
  - `board->camera` 与 `camera->board`
  - `gripper->base` 与 `base->gripper`
- `solvePnP` 的结果在这里按 `board->camera` 使用。
- OpenCV `calibrateHandEye` 的喂法对方向非常敏感，当前 `solve_eye_to_hand.py` 里的约定不要随手改。
- 最新时间的 `session` 不一定质量最好。
  - 采样时抖动、夹持不稳、板子没完整识别，都可能让最新一批更差。
- 验证通过只代表“手眼链路自洽”。
  - 不代表后续抓取一定准确，抓取还会受到 3D 检测中心定义、桌面高度解释、控制链执行状态影响。

## 后续开发建议

- 先看 `validate_eye_to_hand.py` 的误差，再决定是否替换当前外参。
- 如果抓取点明显偏移，优先排查：
  1. `center_camera_xyz_m` 是否合理
  2. `T_base_camera` 是否来自当前可信 session
  3. 抓取脚本是否用了错误的外参文件
- 如果只想验证某批样本是否靠谱，先跑 validate，不要直接拿来抓。
